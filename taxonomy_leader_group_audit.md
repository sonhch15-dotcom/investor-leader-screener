# 미국 업종 분류와 Leader2 구조 감사

구조 실험 Run ID: `us-taxonomy-structure-frozen-20260712-v1`
판정: `exploratory_structure_audit`

## 결론

기존 57개 라벨은 공식 업종 체계로 유지할 수 없다. 이 값은 S&P 500의 넓은 GICS 섹터와 Nasdaq-100 표의 세부 업종명이 수집 순서에 따라 한 필드에 섞인 결과다. 과거 성과가 높았다는 이유로 이 구조를 다시 공식화하면 분류표 자체를 과거 수익률에 맞추는 문제가 생긴다.

Morningstar 일관 분류는 장기 누적 수익이 낮았지만, 혼합 분류보다 최대 하락이 크게 얕았다. 따라서 "Morningstar가 실패했다"가 아니라 "기존 전략 성과 중 분류 착시가 얼마나 컸는지 드러났다"고 해석한다.

운용 결론은 다음과 같다.

1. Morningstar `sector`와 `industryGroup`을 분리한 일관 분류를 향후 연구 기준선으로 유지한다.
2. 산업그룹 표본 보정 8종목 방식은 다음 PIT 검증의 1순위 후보로 둔다.
3. 이번 결과만으로 Public API, 월 추천, Android 주문·알림을 바꾸지 않는다. 현재 추천 엔진이 Morningstar 방식으로 전환됐다는 뜻이 아니다.
4. 기존 57개 라벨은 재현용 `legacy` 필드로만 보존하고 신규 전략의 공식 분류로 사용하지 않는다.

## 57개 라벨의 구조

저장된 517개 미국 주식에서 확인한 값은 다음과 같다.

| 항목 | 결과 |
|---|---:|
| 전체 라벨 | 57개 |
| 넓은 섹터 라벨 | 11개 |
| 세부 업종형 라벨 | 46개 |
| 라벨당 중앙 종목 수 | 1종목 |
| 2종목 이하 라벨 | 40개 |
| 8종목 미만 라벨 | 44개 |
| 가장 작은 라벨 | 1종목 |
| 가장 큰 라벨 | 75종목 |

현재 주도 업종 계산은 최소 3종목을 요구한다. 따라서 57개 중 40개는 업종 순위 경쟁에 처음부터 참여하지 못한다. 평가 가능한 17개 그룹도 금융 75종목, 산업재 71종목 같은 넓은 섹터와 전자부품 4종목, 바이오 6종목 같은 세부 업종이 같은 점수식으로 경쟁한다.

소스 코드상 생성 과정은 다음과 같다.

1. S&P 500 Wikipedia 표에서 `GICS Sector`를 읽는다.
2. Nasdaq-100 표에서 `GICS Sector|Sector`에 맞는 열을 읽는다.
3. 중복 종목은 나중에 읽은 Nasdaq 분류가 앞선 값을 덮는다.

따라서 이 57개는 의도적으로 설계한 계층형 분류가 아니다.

핵심은 절대 수익률이 아니다. 현재 구성 종목 고정이라 생존편향이 있기 때문이다. 같은 입력에서 작은 그룹 처리만 바꿨는데 누적 수익이 +919.5%에서 +423.7%로 줄었다는 민감도가 핵심이다.

원형에서 전자부품 4종목 그룹은 25회 선택돼 가장 자주 등장했다. 8종목 미만 그룹을 제외하면 2025~2026 구간 수익은 +335.0%에서 +75.8%로 낮아졌다. 표본 보정을 적용하면 같은 구간은 +204.0%, MDD는 -22.0%에서 -18.0%로 완화됐다.

## 다음 PIT 검증 계약

한 번에 업종 그룹 처리만 바꾼다.

1. Morningstar 산업그룹 원형
2. 산업그룹 표본 보정 강도 8
3. 유효 종목 5개 미만이면 상위 섹터로 올리는 적응형 그룹
4. 업종 단계 없이 개별 점수 상위 2종목을 고르는 대조군

고정 항목:

- 당시 SPY·QQQ 구성 종목 합집합
- 구성 정보 5거래일 지연
- 신호 다음 1거래일 매수
- 매수·매도 각각 10bp와 25bp 스트레스
- 같은 자금 배분
- 6개월 50% 매도, 잔여분 주봉 연장, 최대 12개월

평가 구간:

- 설계: 2010~2018
- 검증: 2019~2022
- 최근 확인: 2023~사용 가능 마지막 날짜

전체 누적 수익 하나로 승자를 고르지 않는다. 세 구간의 방향, MDD, 36개월·60개월 QQQ 초과 빈도, 상위 수익 lot 의존도, 그룹 크기와 매수 누락을 함께 본다.

## 월간 분류 운영 규칙

1. 월말 신호 생성 전에 분류 스냅샷을 고정한다.
2. 종목별 `sector`, `industryGroup`, 제공처 코드, 적용 시작일과 종료일을 저장한다.
3. 이전 달과 달라진 종목만 검토한다.
4. 분류 버전과 전체 파일 SHA-256을 신호 패키지에 기록한다.
5. 확정된 과거 신호의 분류를 소급 수정하지 않는다.
6. 제공처 분류가 없으면 임의 세부 업종을 만들지 않고 상위 섹터로 보수적으로 올린다.
7. 전략 점수 최적화와 분류 변경 승인을 같은 단계에서 하지 않는다.
