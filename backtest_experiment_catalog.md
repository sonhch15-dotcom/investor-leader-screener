# 백테스트 실험 카탈로그

작성일: 2026-07-13
기계 판독 원본: `config/backtest-experiment-catalog.json`

## 목적

이 문서는 새 아이디어를 실행하기 전에 **같은 가설을 이미 시험했는지** 확인하기 위한 마스터 목록이다. 좋은 결과뿐 아니라 실패, 철회, 엔진 오류가 있었던 실험도 남긴다. 결과가 무효라는 이유로 기록을 지우면 같은 아이디어와 같은 오류를 다시 시험하게 되기 때문이다.

이 카탈로그는 성과 보고서가 아니다. 성과 수치는 각 원본 보고서에서 확인하고, 여기서는 다음 네 가지를 판단한다.

1. 무엇을 이미 시험했는가
2. 그 결과를 어느 정도 믿을 수 있는가
3. 같은 실험을 다시 해도 되는가
4. 다음 실험에서 정말 새로 바뀌는 것은 무엇인가

## 상태 읽는 법

| 상태 | 의미 |
|---|---|
| `retained_rule` | 현재 규칙으로 유지하지만 더 강한 PIT·전진 검증은 별도로 필요할 수 있음 |
| `rejected` | 정상 검증했지만 승격 기준을 통과하지 못함 |
| `invalidated` | 엔진·데이터 오류로 성과 수치를 전부 폐기함. 아이디어 시도 이력만 유효 |
| `superseded` | 더 좋은 데이터 계약의 후속 실험이 있어 이전 결과를 승격 근거로 사용하지 않음 |
| `exploratory_only` | 구조 탐색용이며 공식 성과 근거가 아님 |
| `provisional` | 자료는 남기지만 생존편향이나 상품 이력 문제를 해결하지 못함 |

## QQQ 아이디어 중복 지도

아래 아이디어는 이미 정상 검증됐거나, 최소한 실행을 시도한 기록이 있다.

| 아이디어 | 기록 | 현재 판정 |
|---|---|---|
| 3·6·12-1개월 및 혼합 모멘텀 | 최초 Stage 1~3 | 시도했으나 최초 결과는 엔진 오류로 무효 |
| 상승 경로의 매끄러움 | 최초 `SMOOTH` 점수 | 시도 이력 있음. 완전히 새로운 아이디어로 부르면 안 됨 |
| 저변동성·역변동성 비중 | 최초 Stage 1~3 및 교정 위험 연구 | 반복하지 않음 |
| 시장 베타 제거 잔차 모멘텀 | 최초 `BETA_RESID` | 시도 이력 있음. 최초 성과는 무효 |
| 종목 수 5~20개 | 교정 Stage 1 | 정상 검증 완료 |
| 신규 주도주 30% 혼합 | 교정 `EMERGE` | 정상 검증 완료 |

| 엄격 월교체·순위 완충 | 교정 Stage 2 | 정상 검증 완료 |

| 52주 최고가 근접도 단독 순위 | `qqq-52week-high-proximity-20260713` | 낙폭은 줄었지만 모든 구간에서 QQQ보다 CAGR이 낮아 탈락 |
| 최근 실적 미달 종목 140일 제외 | `qqq-estimize-surprise-veto-20260713` | 개발 구간만 개선되고 검증 구간에서 기존 전략·QQQ에 모두 뒤져 탈락 |
| 실적 예상치 수정 | 같은 연구의 데이터 게이트 | 현재 계정의 Estimize consensus·estimate 이력이 0건이라 성과 실험은 시작하지 못함 |
| 표준화 이익 서프라이즈(SUE) 5분위 | `qqq-sue-signal-validation-20260713` | 자료 관문은 통과했지만 개발·검증을 함께 통과한 3·6·12개월 기간이 없어 탈락 |

따라서 `모멘텀 + 매끄러운 추세 + 일반적인 품질 필터`는 새 가설이 아니다. 정확한 비중만 바꾸는 것은 새로운 경제적 아이디어가 아니라 파라미터 재조합으로 기록한다.

## 아직 시작하지 않은 가설

다음 항목은 코드와 저장 로그에서 실행 기록을 찾지 못했다.

1. `qqq_point_in_time_52week_high_volume_confirmation`
   - 52주 신고가 근접도와 비정상 거래량을 하나의 신규 축으로 시험한다.
   - 52주 최고가 근접도 단독 순위가 먼저 탈락했으므로, 독립적인 경제적 근거와 새 사전등록 없이는 결합 실험을 시작하지 않는다.

`아직 시작하지 않음`은 유망하다는 뜻이 아니다. 단지 중복이 확인되지 않았다는 뜻이다.

## 새 백테스트 시작 규칙

새 연구는 `npm run study:new -- <id> "<이름>" <시장>`으로 만든 뒤 `study.json`의 `noveltyContract`를 먼저 작성한다.

필수 항목:

- `priorArtReviewed`: 이 문서와 JSON 카탈로그를 확인했는가
- `primaryConceptKey`: 이번 연구가 검증하는 중심 가설
- `primaryExperimentFingerprint`: 유니버스·신호·선정·배분·매도 규칙을 연결한 고유 지문
- `nearestPriorExperimentIds`: 가장 비슷한 기존 실험
- `novelDimension`: 기존 실험에서 달라지는 단 하나의 핵심 요소
- `rerunOf`, `rerunReason`: 재현·엔진 수정·새 데이터 때문에 반복하는 경우의 근거

실행 전 검증:

```powershell
npm run study:verify
```

검증기는 다음을 차단한다.

- 이미 사용한 실험 ID
- 이미 사용한 중심 가설
- 이미 사용한 실험 지문
- 과거 실험 검토 없이 시작하는 연구
- 신규 요소를 설명하지 못한 연구
- 근거 없는 동일 실험 재실행

동일 실험을 허용하는 사유는 `reproduction`, `engine_fix`, `new_independent_data`, `cost_stress` 네 가지뿐이다. 이 경우에도 원 실험 ID를 반드시 연결한다.

## 기록 원칙

1. 결과가 좋지 않아도 삭제하지 않는다.
2. 오류가 난 결과는 `invalidated`로 남기고 수치만 증거에서 제외한다.
3. 데이터 계약이 달라지면 같은 전략명이더라도 별도 실험 ID와 지문을 사용한다.
4. 기존 아이디어 여러 개의 비중만 바꾸는 조합은 `새 가설`이 아니라 `기존 가설 재조합`으로 표시한다.
5. 최종 홀드아웃을 열었으면 다시 잠금 구간으로 부르지 않는다.
6. 유효한 결과만 공개 보고서에 싣되, 카탈로그에는 모든 시도 이력을 남긴다.
