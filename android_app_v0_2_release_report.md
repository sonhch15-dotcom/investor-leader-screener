## 1. 릴리즈 목표

v0.2의 목표는 기존 단일 예수금 초안을 `3계좌 전략 운용 콘솔`로 바꾸는 것이다. 이번 APK는 최종 출시형 아키텍처인 Compose/Room까지 전환한 것은 아니지만, 실제 사용 흐름과 화면 구성을 검증할 수 있도록 계좌 중심 UX를 먼저 반영했다.

## 2. 반영된 핵심 변화

### 2.1 계좌 구조

기본 계좌를 아래 3개로 재구성했다.

| 계좌 | 기준 통화 | 목적 |
|---|---:|---|
| 미국 주식 계좌 | USD | 미국 주식 전략 운용 |
| 한국 주식 계좌 | KRW | 한국 주식 전략 운용 |
| 연금 ETF 계좌 | KRW | 한국 ETF 리밸런싱 운용 |

미국 주식 계좌는 USD 주 운용 현금과 KRW 보조 현금을 함께 표시한다.
계좌명은 사용자가 계좌 화면에서 직접 바꿀 수 있다.

### 2.2 화면 구조

하단 탭을 아래 구조로 변경했다.

| 탭 | 역할 |
|---|---|
| 오늘 | 계좌별 Action Inbox, 시장 시간 순서, 총자산 요약 |
| 계좌 | 계좌별 현금, 보유 평가, 투자/현금 비중 |
| 운용 | 이번 달 신호, 주문 가이드, 보유 lot |
| 자산 | 총자산, 계좌별 자산, 손익 분해 |
| 기록 | 빠른 기록, 타임라인, 데이터 동기화, 백업 안내 |

설정 기능은 별도 탭에서 제거하고, 실제 사용 흐름을 방해하지 않도록 `기록` 탭 하단에 배치했다.

### 2.3 주문/기록 UX

주문 흐름은 아래처럼 바꿨다.

```text
전략 신호
  -> 계좌별 주문 가이드
  -> 사용자가 증권사 앱에서 주문
  -> 실제 수량/평균단가/수수료 입력
  -> 목표 금액 대비 검증
```

검증 기준:

- 목표 금액 대비 5% 이내면 완료
- 차이가 최소 주문 단위 이내면 현실적 완료
- 부족/초과 금액이 크면 조정 필요 표시

### 2.4 디자인 테마

디자인팀 피드백 반영:

- 중립 배경과 흰색 카드 기반의 차분한 자산관리 앱 톤
- 계좌별 낮은 채도 배지 색상
- 카드 반경 8dp 기준
- 금액은 더 굵고 읽기 쉽게 표시
- 하단 Android 시스템 버튼과 겹치지 않도록 navigation bar inset 반영
- 자산/현금 비중을 막대로 시각화

### 2.5 데이터 동기화

기존 데이터 연동은 유지했다.

- `signals/latest.json`
- `weekly-trends/latest.json`
- `prices/latest.json`
- `fx/latest.json`

원격 API URL이 저장되어 있으면 앱 실행 시 조용히 자동 동기화한다.

## 3. 팀별 리뷰 결과

### UX팀

좋아진 점:

- 첫 화면에서 총자산, 시장 순서, 계좌별 할 일이 함께 보인다.
- 계좌별 ActionCard로 어떤 계좌의 작업인지 명확해졌다.
- 설정 탭을 제거해 매일 쓰는 탭이 더 실전 운용 중심이 되었다.

남은 과제:

- 주문 실행 세션이 아직 완전하지 않다.
- 입력 오류 수정/되돌리기 UX가 부족하다.
- 주문 실행 세션과 기록 되돌리기 UX가 아직 완전하지 않다.

### 디자인팀

좋아진 점:

- 초안보다 정보 위계가 명확해졌다.
- 하단 내비게이션 안전 영역 문제가 개선됐다.
- 계좌별 색상과 자산 비중 막대로 스캔성이 좋아졌다.

남은 과제:

- 진짜 출시 수준으로 가려면 Compose/Material 3 전환이 필요하다.
- 다크 모드와 큰 글자 모드 QA가 아직 부족하다.
- 차트는 아직 정교한 그래프가 아니라 비중 막대 중심이다.

### 자산운용팀

좋아진 점:

- 계좌별 현금과 보유가 분리됐다.
- 총자산은 통합 계산된다.
- 원금 목표 대비 매수 완료 검증이 들어갔다.
- ETF는 연금 계좌 중심으로 배치됐다.

남은 과제:

- TWR/MWR 성과 계산은 아직 없다.
- 월말 스냅샷과 기준지수 대비 성과 차트는 다음 단계다.
- 환율 효과 분해는 표시 구조만 있고 계산 엔진은 아직 단순하다.

### 검증팀

확인한 것:

- Android debug 빌드 성공
- APK 안에 신호, 가격, 환율 asset 포함 확인
- 하단 navigation bar inset 처리 코드 반영
- versionCode 11, versionName 0.2.9 적용

남은 QA:

- 실제 기기에서 작은 화면/큰 글자/3버튼 내비게이션 스크린샷 확인
- 장시간 사용 후 장부 누적 테스트
- 입력 실수 수정 플로우 테스트

## 6. v0.2.1 UX 패치

사용자 실기기 확인 후 아래 불편 사항을 수정했다.

- 입금 기록에서 통화 입력칸을 제거하고 `KRW`, `USD` 선택 버튼으로 변경
- 입금 기록에 빠른 금액 버튼 추가
- 체결 기록의 맨 아래 전략 키 입력칸 제거
- 전략명은 체결 폼 상단에 읽기 전용 정보로 표시
- 체결 기록은 수량, 평균단가, 비용, 선택 메모만 입력
- 매도 기록에 `50% 수량`, `전량` 버튼 추가
- 운용 화면을 미국 주식, 한국 주식, 연금 ETF 섹션으로 완전히 분리
- 각 섹션에서 현재 전략을 확인할 수 있도록 변경

## 7. v0.2.2 운용 UX 패치

추가 사용자 피드백 후 아래 내용을 반영했다.

- 운용 화면을 `미국`, `한국`, `ETF` 미니탭 전환 구조로 변경
- `이 전략 잠시 중지` 버튼 제거
- 각 미니탭에 `전략 바꾸기` 버튼 추가
- 전략 선택 목록에서 우리가 만든 active/research 전략을 선택 가능하게 변경
- 미국 기본 전략명을 `Leader2 + Repeat Theme Combo Cap27.5`로 정확히 표시
- 현재 추천 신호는 `us_leader2_repeat_theme_combo_cap27_5` 기준임을 확인
- 연구 전략을 선택하면 현재 신호 패키지 확장이 필요하다는 안내 표시
- 입금/출금 금액 입력 시 `1,000,000` 형식 자동 콤마 처리
- 전반적인 버튼 높이와 카드 여백 축소

## 8. v0.2.3 입력 UX 정리

- 입금/출금 다이얼로그에서 빠른 금액 버튼 제거
- 금액은 직접 입력하되 `1,000,000` 형식 자동 콤마만 유지

## 9. v0.2.4 계좌/차트 UX 패치

- 계좌 화면에 `전체`, `미국`, `한국`, `ETF` 미니탭 추가
- 전체 계좌 화면에 계좌별 자산 비중 도넛 차트 추가
- 개별 계좌 화면에 현금/보유 구성 도넛 차트 추가
- 계좌 또는 전체 자산에 보유 종목이 있으면 보유 종목 비중 도넛 차트 표시
- 자산 화면에 총자산 계좌 비중과 보유 종목 비중 도넛 차트 추가
- 외부 차트 라이브러리 없이 Android 커스텀 View로 `DonutChartView` 구현

## 10. v0.2.5 차트 표시 보정

- 도넛 차트가 카드 안에서 상하단으로 잘리지 않도록 높이, 반지름, stroke 계산을 조정
- 표시할 자산 비중이 없는 빈 차트는 compact 모드로 축소 표시
- APK 버전: versionCode 7, versionName 0.2.5

## 11. v0.2.6 미국 전략 주문금액 보정

- 미국 `Leader2 + Repeat Theme Combo Cap27.5` 주문 가이드가 가용 현금을 2개 종목에 단순 균등분할하던 문제 수정
- 계좌 평가 원금 기준 기본 매수 비율과 종목당 27.5% 상한을 적용
- 반복 매수 이력과 AI 하드웨어 테마 가중을 목표 주문금액 계산에 반영
- APK 버전: versionCode 8, versionName 0.2.6

## 12. v0.2.7 한국 주식/ETF 전략 보정

- 한국 주식 `Leader2` 주문금액이 단순 2분할로 계산되던 문제 수정
- 원 전략의 capital-account sizing: 초기 3개월 월 30%, 이후 월 15%, 종목당 22.5% 상한 반영
- ETF `Core/Satellite/Defense 50/40/10`은 목표비중 표시에서 실제 매수/매도 수량 가이드로 확장
- APK 버전: versionCode 9, versionName 0.2.7

## 13. v0.2.8 업데이트 캐시 보정

- 기존 앱 데이터에 남은 원격 신호 캐시가 새 APK 내장 데이터보다 우선 적용되던 문제 수정
- APK 업데이트 후 asset packageVersion이 더 최신이면 자동으로 캐시를 비우고 새 전략 패키지를 로드
- APK 버전: versionCode 10, versionName 0.2.8

## 14. v0.2.9 오래된 원격 동기화 차단

- 자동 원격 동기화가 APK 내장 데이터보다 오래된 패키지를 다시 저장하지 못하도록 차단
- 원격 데이터가 오래되면 내장 데이터 상태를 유지
- APK 버전: versionCode 11, versionName 0.2.9
