# Android App Strategy Product Report

작성일: 2026-07-09

## 1. 회의 목적

지금까지 만든 전략을 웹 대시보드에서 보는 수준을 넘어, 사용자가 실제 투자 과정에서 바로 활용할 수 있는 Android 앱으로 발전시키기 위한 제품/알고리즘 기획안이다.

핵심 목표는 다음과 같다.

- 초보 투자자도 전략 기반 매매를 따라 할 수 있게 만든다.
- 매수, 매도, 보유 연장, 리밸런싱 판단을 앱이 날짜와 금액 기준으로 안내한다.
- 입금, 출금, 환율, 수수료, 세금, 보유 종목 중복, 현금 부족 같은 현실 조건을 알고리즘에 반영한다.
- 미국 주식, 한국 주식, 한국 ETF 전략을 같은 구조에서 운용하되 전략별 차이는 명확히 분리한다.
- 향후 새 전략을 추가해도 같은 템플릿으로 누적 검증, 운용, 알림까지 연결되게 만든다.

## 2. 협업 팀 구성

### 2.1 UX 팀

역할: 투자자가 앱을 열었을 때 가장 먼저 해야 할 행동을 쉽게 이해하도록 흐름을 설계한다.

결론:

- 첫 화면은 백테스트 설명보다 `오늘 할 일` 중심이어야 한다.
- 사용자는 "왜 이 종목인지"보다 먼저 "지금 사야 하는지, 팔아야 하는지, 얼마를 넣어야 하는지"를 알고 싶어 한다.
- 과거 추천 종목은 투자 시작 전 화면에 기본 매수 대상으로 보여주면 혼란스럽다.
- 투자 시작일 이후의 신호만 실제 매수 가이드로 사용해야 한다.

### 2.2 전략/퀀트 팀

역할: 현재 공식 전략과 테스트 전략을 구분하고, 앱에서 운용 가능한 규칙으로 변환한다.

결론:

- 미국 공식 전략은 `Leader2 + Repeat Theme Combo Cap27.5`를 기본값으로 둔다.
- Cap30은 성과는 가장 높았지만 집중도가 높으므로 공격형 참고 전략으로 분리한다.
- Conviction Diverse Top2는 현재 공식 전략을 대체하지 않고 `testing` 상태로 보관한다.
- 한국 주식과 한국 ETF는 미국 전략과 화면 템플릿은 통일하되 매매 규칙은 별도 전략 객체로 관리한다.

### 2.3 포트폴리오/자금배분 팀

역할: 실제 자본금 안에서 얼마를 매수하고, 현금 부족과 중복 보유를 어떻게 처리할지 정의한다.

결론:

- 계좌는 최소 3개로 분리한다: 미국 주식, 한국 주식, 한국 ETF/연금.
- 종목별 한도는 개별 매수 금액이 아니라 전체 보유 원금 기준으로 계산한다.
- 같은 종목이 여러 달 반복 추천되면 하나의 종목 카드 안에 여러 매수 회차를 보여준다.
- 각 회차는 독립된 lot으로 보관하며, 6개월 50% 매도와 나머지 주봉 연장 매도를 각각 추적한다.

### 2.4 매매 실행/알림 팀

역할: 전략 신호를 실제 주문 전 체크리스트와 알림으로 바꾼다.

결론:

- 월말 확정 신호, 다음 거래일 매수, 6개월 50% 매도, 주봉 훼손 매도, ETF 리밸런싱을 각각 알림 이벤트로 만든다.
- 앱은 자동 주문을 전제로 하지 않는다. 사용자가 실제 주문 후 체결가와 수량을 기록해야 한다.
- 기록된 실제 체결 정보가 이후 수익률, 매도 예정일, 세금/수수료 계산의 기준이 된다.

### 2.5 계좌/회계/현실 비용 팀

역할: 입금, 출금, 환율, 수수료, 세금, 배당, 거래 단위 같은 현실 변수를 처리한다.

결론:

- 원화와 달러를 분리해서 기록하고, 평가액은 최신 환율로 환산한다.
- 매수/매도 시점의 환율은 거래 기록에 고정 저장한다.
- 수수료와 세금은 브로커별 설정값으로 관리하고, 백테스트와 실제 계좌 모두 같은 비용 엔진을 사용한다.
- 소수점 거래 가능 여부를 계좌 설정으로 두고, 불가능하면 남는 금액은 현금으로 남긴다.

### 2.6 데이터/백엔드 팀

역할: 로그인, 데이터 저장, 가격 업데이트, 알림 스케줄러를 설계한다.

결론:

- 현재 웹의 localStorage만으로는 최종 앱 목표에 부족하다.
- 사용자의 자본금, 거래 기록, 알림 설정, 계좌 구분은 로그인 기반 DB에 저장해야 한다.
- 매월 신호 생성과 매주 보유 점검은 서버 작업으로 돌리고, 결과만 앱에 전달하는 구조가 적합하다.
- 데이터 오류나 가격 갱신 실패가 있으면 이전 신호를 유지하고 `데이터 지연` 경고를 띄워야 한다.

### 2.7 Android/PWA 팀

역할: 웹에서 Android 앱까지 가는 구현 단계를 정한다.

결론:

- 바로 네이티브 Android로 넘어가기보다 PWA 전단계 기능을 먼저 완성하는 것이 좋다.
- 1차는 웹/PWA에서 로그인, 포트폴리오, 알림 모델, 전략 실행 흐름을 검증한다.
- 2차는 Android 설치형 앱으로 전환한다. 후보는 TWA, Capacitor, React Native, Kotlin/Compose다.
- 현재 앱이 웹 기반이므로 초기 Android화는 TWA 또는 Capacitor가 현실적이다.

### 2.8 리스크/컴플라이언스 팀

역할: 투자 앱으로서 오해와 위험을 줄인다.

결론:

- 앱은 투자 조언 자동 실행기가 아니라 전략 기반 의사결정 보조 도구로 표시해야 한다.
- 모든 매수/매도는 사용자의 최종 확인이 필요하다.
- 백테스트는 미래 수익을 보장하지 않는다는 안내가 필요하다.
- 가격 데이터 지연, 세금 계산 단순화, 환율 오차 가능성을 명확히 표시해야 한다.

## 3. Android 앱 핵심 기능

### 3.1 오늘 할 일

앱 첫 화면에서 가장 먼저 보여줄 내용이다.

- 이번 달 신규 매수 후보
- 매수할 금액
- 매수 예정일 또는 다음 거래일
- 50% 매도 예정 종목
- 주봉 훼손으로 나머지 물량 정리 검토가 필요한 종목
- ETF 리밸런싱 필요 여부
- 현금 부족, 비중 초과, 환율 급변, 데이터 지연 경고

### 3.2 투자 시작 마법사

처음 투자하는 사용자는 아래 순서로 시작한다.

1. 투자 시장 선택: 미국 주식, 한국 주식, 한국 ETF
2. 계좌 선택 또는 생성
3. 시작 자본 입력
4. 통화 선택: KRW, USD
5. 환율 적용 방식 선택: 자동 환율, 수동 입력
6. 수수료/세금 프로필 선택
7. 소수점 거래 가능 여부 선택
8. 전략 선택: 기본은 공식 전략
9. 현재 월의 매수 후보와 금액 확인
10. 실제 주문 후 체결가와 수량 기록

중요 규칙:

- 투자 시작일 이전 추천 종목은 매수 대상으로 가져오지 않는다.
- 사용자가 2026년 7월에 시작하면 2026년 7월 확정 후보부터 운용한다.
- 과거 추천 종목은 백테스트와 참고 기록에만 표시한다.

### 3.3 전략 라이브러리

전략은 아래 상태로 관리한다.

- `active`: 실제 운용 전략
- `testing`: 백테스트와 관찰 중인 전략
- `paused`: 일시 중단된 전략
- `retired`: 더 이상 사용하지 않는 전략

현재 기본 전략:

| 전략 | 시장 | 상태 | 기준지표 | 핵심 규칙 |
|---|---|---|---|---|
| Leader2 + Repeat Theme Combo Cap27.5 | 미국 주식 | active | QQQ | 월간 주도 섹터 2개에서 1종목씩, 반복/AI 하드웨어 가중, 종목 한도 27.5% |
| KR Stock Leader2 | 한국 주식 | active | KOSPI200 | 한국 우량주 유니버스에서 월간 주도 업종 2개 대표 종목 |
| KR ETF Core Satellite | 한국 ETF | active | KOSPI200 ETF | 코어 50%, 위성 40%, 방어 10% 월간 리밸런싱 |
| Conviction Diverse Top2 | 미국 주식 | testing | QQQ | 후보 확신도와 섹터 분산을 결합한 테스트 전략 |

### 3.4 매수 가이드

미국 공식 전략 기준:

- 월말 기준으로 후보를 확정한다.
- 다음 거래일에 매수 가이드를 만든다.
- 월중 순위 변화는 관찰 정보로만 표시하고, 이미 확정된 매수 후보를 자동 변경하지 않는다.
- 월간 주도 섹터 상위 2개에서 각각 1종목을 선택한다.
- 반복 추천 종목, AI/반도체 하드웨어 관련 종목에는 매수 금액 가중치를 준다.
- 종목별 누적 원금 한도는 27.5%다.
- 한도 초과 시 매수 금액을 줄이거나 스킵한다.

### 3.5 매도 가이드

미국 공식 전략 기준:

- 매수 후 6개월이 지나면 해당 lot의 50%를 기본 매도한다.
- 나머지 50%는 주봉 추세가 유지되면 연장 보유한다.
- 주봉 10주선 2주 연속 이탈 또는 주봉 RSI 50 하회가 확인되면 나머지 물량을 매도 검토한다.
- 최대 12개월이 되면 남은 물량을 정리한다.
- 모든 매도는 lot 단위로 관리한다.

표시 방식:

- 종목 카드 1개 안에 추천월별 lot을 리스트로 표시한다.
- 각 lot에는 매수일, 매수가, 수량, 현재 수익률, 50% 매도일, 잔여 물량 상태, 최종 매도 조건을 표시한다.
- 종목 전체 카드에는 전체 투자 원금, 평가액, 실현손익, 미실현손익, 종합 수익률을 표시한다.

### 3.6 한국 ETF 리밸런싱

한국 ETF 전략은 종목 매수/매도보다 계좌 비중 관리가 핵심이다.

- 매월 말 ETF 후보와 목표 비중을 확정한다.
- 다음 거래일에 현재 보유 비중과 목표 비중을 비교한다.
- 목표 비중보다 낮은 ETF는 추가 매수한다.
- 목표 비중보다 높은 ETF는 일부 매도한다.
- 리밸런싱 후 현금이 너무 많이 남으면 최소 주문 단위에 맞춰 다시 배분한다.
- 연금 계좌는 매매 가능 ETF와 세금 처리가 다를 수 있으므로 별도 계좌 설정을 둔다.

## 4. 현실 조건 처리 알고리즘

### 4.1 입금 처리

입금이 발생하면 다음 순서로 처리한다.

1. 해당 계좌의 현금을 증가시킨다.
2. 과거 추천월의 종목을 뒤늦게 매수하지 않는다.
3. 다음 월간 신호부터 신규 자금에 반영한다.
4. 입금액이 크면 1개월에 모두 투입하지 않고 2~3개월 램프 투입 옵션을 제공한다.
5. 종목별 27.5% 한도와 계좌별 현금 예비율은 유지한다.

예외:

- 사용자가 `즉시 투입`을 선택하면 현재 월 공식 후보 안에서만 추가 매수한다.
- 한도 초과 종목은 매수하지 않고 다음 후보 또는 현금으로 남긴다.

### 4.2 출금 처리

출금이 발생하면 다음 순서로 처리한다.

1. 현금에서 먼저 차감한다.
2. 현금이 부족하면 매도 예정이 가까운 lot부터 정리한다.
3. 그다음 주봉 훼손 경고가 있는 연장 보유 lot을 정리한다.
4. 그다음 비중 초과 종목을 줄인다.
5. 그래도 부족하면 사용자가 직접 매도 대상을 선택하게 한다.

중요:

- 전략상 매도 시점이 아닌 종목을 앱이 자동으로 강제 매도하지 않는다.
- 긴급 출금은 전략 성과와 별도로 기록한다.

### 4.3 환율 처리

미국 주식 계좌는 원화 기준 사용자에게 환율 영향이 크다.

기본 원칙:

- 매수 시점 환율을 lot에 저장한다.
- 매도 시점 환율을 거래 기록에 저장한다.
- 현재 평가액은 최신 환율로 환산한다.
- 달러 기준 수익률과 원화 기준 수익률을 분리해 보여준다.

알림:

- 환율이 사용자가 정한 기준 이상 변동하면 환율 경고를 보낸다.
- 환전이 필요한 매수 가이드에서는 필요한 원화 금액과 예상 달러 금액을 함께 표시한다.

### 4.4 수수료와 세금 처리

모든 전략 계산은 비용 차감 전/후를 분리한다.

수수료 프로필:

- 미국 주식: 매매 수수료, 환전 스프레드, SEC/TAF 등 매도 비용, 배당 원천징수
- 한국 주식: 매매 수수료, 거래세, 제세금
- 한국 ETF: 매매 수수료, ETF 과세 유형, 연금 계좌 과세 방식

앱 처리:

- 사용자는 증권사별 기본 프로필을 선택하거나 직접 입력한다.
- 백테스트는 기본 비용 가정과 사용자 비용 설정을 모두 보여줄 수 있어야 한다.
- 실제 계좌 성과는 사용자가 기록한 실제 수수료가 우선한다.

### 4.5 소수점 거래와 최소 주문

처리 규칙:

- 소수점 거래 가능 계좌는 목표 금액에 최대한 맞춰 매수한다.
- 정수 주식만 가능한 계좌는 살 수 있는 최대 주식 수를 계산하고 남는 금액은 현금으로 남긴다.
- 최소 주문 금액보다 작은 매수 신호는 스킵하거나 다음 달 현금으로 이월한다.

### 4.6 중복 추천 종목

같은 종목이 여러 달 반복 추천될 수 있다.

처리 규칙:

- 종목 카드는 하나로 합친다.
- 카드 안에 추천월별 lot을 별도로 쌓는다.
- 종목 전체 한도는 모든 lot의 원금을 합산해 계산한다.
- 6개월 50% 매도와 주봉 연장 매도는 lot별로 따로 진행한다.

### 4.7 배당, 분할, 티커 변경

앱은 아래 이벤트를 별도 기록으로 처리한다.

- 배당: 현금 증가로 기록하고 성과에는 포함한다.
- 주식분할: 수량과 단가를 조정하되 총 원금은 유지한다.
- 티커 변경: 기존 lot을 새 티커로 연결한다.
- 상장폐지/합병: 사용자가 수동 조정할 수 있는 이벤트 처리 화면을 둔다.

### 4.8 데이터 오류

데이터가 불완전하면 앱은 새 신호를 만들지 않는다.

처리 규칙:

- 일부 가격 데이터가 없으면 해당 월 신호 생성 중단
- 이전 신호 유지
- `데이터 지연` 또는 `검증 필요` 표시
- 사용자가 수동으로 강제 적용하려면 별도 확인 필요

### 5.2 매수 금액 계산

```text
buildBuyPlan(account, signal):
  availableCash = account.cash - account.reserveCash
  for candidate in signal.candidates:
    baseWeight = strategy.baseWeight
    multiplier = repeatThemeMultiplier(candidate)
    targetWeight = min(baseWeight * multiplier, strategy.symbolCap)
    maxAllowed = account.equity * strategy.symbolCap - currentSymbolCost(candidate.symbol)
    orderAmount = min(account.equity * targetWeight, availableCash, maxAllowed)
    orderAmount = applyMinOrderAndFractionalRule(orderAmount)
    createOrderGuide(candidate, orderAmount)
```

weeklyHoldingCheck(lot):

## 10. 최종 결론

Android 앱의 핵심은 더 많은 차트와 지표를 보여주는 것이 아니라, 전략을 실제 행동으로 바꿔주는 것이다.

따라서 다음 개발 방향은 다음이 가장 적합하다.

1. 웹에서 먼저 `투자 시작`, `운용 현황`, `알림 규칙`, `자산 관리`를 완성한다.
2. 현재 공식 전략인 `Leader2 + Repeat Theme Combo Cap27.5`를 기본 전략으로 둔다.
3. 한국 주식과 한국 ETF도 같은 앱 흐름에서 운용하되 계좌와 전략 규칙은 분리한다.
4. 입금, 출금, 환율, 수수료, 세금, 소수점 거래, 중복 추천을 알고리즘의 기본 입력값으로 다룬다.
5. PWA 수준까지 안정화한 뒤 Android 앱으로 감싼다.

즉, 지금 당장 Android 네이티브 개발을 시작하기보다, Android 앱이 그대로 사용할 수 있는 투자 실행 엔진을 웹에서 먼저 완성하는 것이 가장 좋은 순서다.
