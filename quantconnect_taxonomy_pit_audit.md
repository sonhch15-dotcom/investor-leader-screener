# 작은 업종을 합치면 전략이 더 좋아질까?

## 결론

`업종 흐름 반영형 + 작은 업종 합치기`는 연구 후보로 보존하되 공식 전략으로 승격하지 않는다.

- 일관 업종 원형보다 누적 수익은 `+35.09%p` 높았다.
- 최대 하락은 `1.46%p` 얕았다.
- 그러나 설계 확인 구간에서는 `-15.37%p` 뒤졌다.
- 최고 수익 lot 두 개를 제외하면 우위는 `+0.45%p`만 남았다.
- QQQ 장기 기회비용, 큰 수익 lot의 종목 식별자, 6~12개월 미래 관찰을 통과하지 못했다.
- Public API, Android 주문·알림, 기존 lot의 청산 일정은 변경하지 않는다.

## 연구 질문

Morningstar 분류 자체가 나쁜 것이 아니라, 종목 수가 너무 적은 업종의 점수가 불안정했던 것은 아닌지 확인했다. 계좌와 매매 규칙은 고정하고 업종을 묶는 방식만 일곱 가지로 바꿨다. 종목 점수는 업종 흐름 반영형과 종목 힘 균형형을 각각 적용해 총 14개 조합을 비교했다.

## 재현성 수정

첫 실행에서는 Python `set` 순회 순서와 점수 동점 처리 순서가 프로세스마다 달라 일부 선택 종목이 바뀌었다. 다음 순서를 고정한 뒤 결과를 다시 만들었다.

1. 종목은 고정 ticker key 순서로 정렬한다.
2. 업종 key를 문자열 순서로 정렬한다.
3. 점수가 같으면 ticker key로 순위를 정한다.
4. 같은 클라우드 코드로 독립 백테스트를 두 번 실행한다.
5. 두 실행의 결과 로그 93줄이 모두 같을 때만 결과를 채택한다.

채택한 QuantConnect backtest ID는 `dcacd8d01b3b98eb3007d95d341fbe0b`, `64173920abc14bd159761164c52376ad`다.

후보의 원래 우위 `+35.09%p`는 최고 두 건을 빼면 `+0.45%p`로 줄었다. 이 검사는 최종 자산에서 이익을 단순 차감한 집중도 스트레스이며 자금을 다시 배분한 재실행은 아니다. 그래도 결론이 한두 건에 매우 민감하다는 사실은 분명하다.

`CEG@2011-09`는 현재 ticker 문자열 표시만으로 2011년의 정확한 증권을 확정하면 안 된다. ticker 재사용이나 symbol mapping 가능성이 있으므로 다음 실행에서 QuantConnect Security Identifier, 상장·인수 기업행위, 실제 체결 가격을 함께 출력해 감사한다.

작은 업종 합치기는 의도한 대로 4종목 이하 그룹 선택을 없앴다. 하지만 월 추천 두 종목 중 원형과 같은 종목은 평균 `1.16개`였고, 두 종목이 모두 같은 달은 `25.0%`뿐이다. 분류 변경이 단순한 점수 보정이 아니라 포트폴리오 구성을 크게 바꾼다는 뜻이다.

같은 작은 업종 합치기에서 종목 점수만 바꾸면 업종 흐름 반영형은 `+248.39%, MDD -18.86%`, 종목 힘 균형형은 `+209.67%, MDD -34.96%`였다. 두 변경을 동시에 적용했을 때 업종 흐름 반영형이 수익 `38.72%p`, MDD `16.10%p` 앞섰다. 이 결과만으로 다른 분류 계약에서 검증한 종목 힘 균형형을 자동 폐기하지는 않는다.

## 시간에 따른 분류 변경

이번 실행에서 Morningstar 분류 변경은 `0건, 0종목`으로 관찰됐다. 이는 분류가 실제로 한 번도 바뀌지 않았다는 증명이 아니다. QuantConnect 문서는 Morningstar US Fundamental Data의 장기 이력과 자산 분류 필드를 제공한다고 설명하지만, 과거 재분류 시점이 모두 완전하게 노출된다고 명시하지는 않는다.

따라서 운용용 분류는 다음처럼 관리한다.

1. 매월 신호 확정 전에 종목, sector, industryGroup, 제공처, 적용 월을 저장한다.
2. 스냅샷 전체에 SHA-256을 붙여 같은 월의 분류를 나중에 바꾸지 못하게 한다.
3. 회사 분할, 인수, 사업 변경으로 달라진 종목만 사람이 확인한다.
4. 백테스트는 그달 저장된 버전을 사용하고 현재 분류로 과거를 덮어쓰지 않는다.
5. 완전한 과거 재분류 이력을 확보하지 못한 기간은 한계로 명시한다.
